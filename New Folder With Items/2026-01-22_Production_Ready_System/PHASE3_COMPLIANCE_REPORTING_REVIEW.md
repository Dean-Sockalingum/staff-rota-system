## Updated Report Template Standards

### **All Compliance Reports Must Include:**

**1. Header Section:**
- Report title
- Organization name and care home(s) covered
- Reporting period (date range)
- Report generation date and time
- Report author/responsible manager
- Distribution list
- Confidentiality statement

**2. Executive Summary (1 page max):**
- Key findings (3-5 bullet points)
- Traffic light status (Green/Amber/Red overall)
- Critical actions required
- Performance vs. targets
- Trend summary (improving/stable/deteriorating)

**3. Quality Theme Alignment:**
- Explicit statement: "This report provides evidence for Care Inspectorate Quality Theme: [Wellbeing/Leadership/Staff/Setting/Care & Support]"
- Specific quality indicators addressed
- HIS QMS principle alignment noted
- NES QI methodology referenced (if applicable)

**4. Data Presentation:**
- Baseline data clearly stated
- Current performance with % change from baseline
- Target/standard reference (regulatory requirement, organizational goal)
- Visual representation (charts, graphs, tables)
- Trend analysis (time-series data showing improvement/deterioration)

**5. Narrative Analysis:**
- What the data means (interpretation)
- Why performance is at this level (contributing factors)
- Comparison to previous periods
- Benchmarking context (peer comparison when available)

**6. Actions and Improvement:**
- Actions taken during reporting period
- Outcomes of previous actions
- Planned actions for next period
- Responsible person and deadline for each action
- Link to PDSA improvement projects (if active)

**7. Risk and Compliance:**
- Compliance status (meeting regulatory requirements?)
- Risks identified from data
- Mitigation measures in place
- Escalation needs (to board, regulators, external partners)

**8. Appendices:**
- Detailed data tables
- Methodology notes (how data collected, calculations)
- Glossary of terms and acronyms
- References to related reports
- Contact information for queries

**9. Footer:**
- Page numbers (Page X of Y)
- Document version control (v1.0, v1.1, etc.)
- Review date (when report template will be reviewed/updated)
- Regulatory framework references (CI, HIS, NES, SSSC)
